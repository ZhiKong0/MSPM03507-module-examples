# MSPM03507各模块例程代码

这个仓库用于持续收集、筛选和整理 MSPM0G3507 / Cortex-M0+ 电赛相关例程代码。

当前阶段是候选池，不直接判断代码质量已可用于比赛。所有外部来源先放入 `待整理的文件夹/`，保留来源、许可证、适配价值和整理状态，后续再拆分为电机、编码器、灰度巡线、IMU、OLED、视觉通信、调参、构建模板等模块。

## 目录

- `待整理的文件夹/`：外部候选代码入口，优先以 Git submodule 方式引用上游仓库。
- `待整理的文件夹/README.md`：候选池总入口，说明现在该先看哪里。
- `待整理的文件夹/00_整理工作台/`：统一整理 goal、来源盘点、模块分类和第一轮拆解路线。
- `待整理的文件夹/CANDIDATES.md`：候选清单、筛选理由、许可证和下一步处理建议。
- `待整理的文件夹/SOURCE_POLICY.md`：导入规则，避免来源和许可证混乱。

## 当前导入原则

1. GitHub 仓库优先作为 submodule 导入，保留上游历史和来源。
2. 有明确许可证的代码可进入后续整理；无许可证代码只作为学习候选，不直接复制合并。
3. 与电赛控制题直接相关的代码优先：差速底盘、编码器 PID、灰度巡线、IMU 航向、OLED 菜单、蓝牙/串口调参、视觉小车、云台控制。
4. 与当前本地工具链偏好一致的工程优先：CMake/Makefile + ARM GCC，其次 Keil/CCS/SysConfig 工程。

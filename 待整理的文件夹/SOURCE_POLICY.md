# 来源导入规则

## 导入方式

- `submodule`：首选方式。适合 GitHub/Gitee 等 Git 仓库，保留完整上游来源。
- `source-card`：适合网页、论坛、压缩包、许可证不明或暂不适合复制的来源，只记录链接和判断。
- `vendored-copy`：只在许可证允许、来源清楚、确实需要脱离上游整理时使用。

## 许可证处理

- MIT/BSD/Apache-2.0：可整理复用，保留 LICENSE 和来源说明。
- GPL：可学习和导入候选，但后续合并到项目时要按 GPL 传染性谨慎处理。
- 无许可证：默认不能直接复制重发布，只能作为 submodule 或来源卡片候选。
- 未确认来源：不得直接合并到正式模块。

## 候选优先级

- A：可直接用于电赛控制题主线，覆盖底盘/编码器/巡线/IMU/菜单/调参等关键模块。
- B：模块价值高，但需要较多移植或许可证注意。
- C：题目特化或框架价值有限，只作为思路参考。

## 官方 SDK 压缩包处理

- TI MSPM0 SDK 这类官方压缩包可作为 `vendored-copy` 候选导入，但必须保留原始版本号、来源路径、license、manifest 和 release notes。
- 不直接提交完整 SDK 大包、工具安装包、IDE 缓存、`.metadata/.tirex` 全量缓存、构建输出或预编译镜像。
- 可导入与目标板强相关的例程目录、源码头文件和最小构建上下文；完整编译仍以本地安装的同版 SDK 为准。

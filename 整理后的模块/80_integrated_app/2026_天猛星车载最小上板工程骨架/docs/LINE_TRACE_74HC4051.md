# 74HC4051 循迹接入说明

本工程当前默认把 8 路循迹从“8 个 GPIO 直读”改成“74HC4051 地址选择 + Z 单线读取”：

- `BOARD_APP_ENABLE_LINE_GPIO=0`
- `BOARD_APP_ENABLE_LINE_MUX_4051=1`
- JY61P 仍然使用 `UART0`，不和 74HC4051 共用信号。

## 接线表

| 74HC4051 引脚 | 接到 MSPM0G3507 / 模块 | 代码宏 | 方向 | 说明 |
|---|---|---|---|---|
| `VCC` | `3.3V` | - | 电源 | 推荐 4051 和 MCU 同为 3.3V 逻辑域。 |
| `GND` | `GND` | - | 电源 | 必须和循迹板、MSPM0 共地。 |
| `VEE` | `GND` | - | 电源 | 单电源数字信号使用时接地。 |
| `S0` | `PB0` | `LINE_MUX_S0_PIN` | MCU 输出 | 地址最低位。 |
| `S1` | `PB1` | `LINE_MUX_S1_PIN` | MCU 输出 | 地址中间位。 |
| `S2` | `PB10` | `LINE_MUX_S2_PIN` | MCU 输出 | 地址最高位。 |
| `E` | `PB14` 或直接接 `GND` | `LINE_MUX_E_PIN` | MCU 输出/固定低 | 低电平使能；当前代码默认拉低。 |
| `Z` | `PB11` | `LINE_MUX_Z_PIN` | MCU 输入 | 8 路循迹复用后的单线输出。 |

| `Y2` | 循迹 `CH2` | - | 4051 输入 | - |
| `Y3` | 循迹 `CH3` | - | 4051 输入 | - |
| `Y4` | 循迹 `CH4` | - | 4051 输入 | - |
| `Y5` | 循迹 `CH5` | - | 4051 输入 | - |
| `Y6` | 循迹 `CH6` | - | 4051 输入 | - |
| `Y7` | 循迹 `CH7` | - | 4051 输入 | - |

## 选通顺序

| `S2 S1 S0` | 被选中的 4051 通道 | 写入 `raw_bits` |
|---|---|---|
| `000` | `Y0` | bit0 |
| `001` | `Y1` | bit1 |
| `010` | `Y2` | bit2 |
| `011` | `Y3` | bit3 |
| `100` | `Y4` | bit4 |
| `101` | `Y5` | bit5 |
| `110` | `Y6` | bit6 |
| `111` | `Y7` | bit7 |

代码每轮循迹采样都会依次设置 `S0/S1/S2`，等待 `BOARD_APP_LINE_MUX_SETTLE_CYCLES`，然后读取 `Z`，最后拼成 8 位 `raw_bits` 交给原来的循迹算法。

## 和 UART 串行方案的性能差别

74HC4051 不是 UART 串口芯片，它只是 8 选 1 模拟/数字多路复用器。它的“串行”含义是 MCU 用地址线逐个选择通道，再从一根 `Z` 线上读电平。

| 方案 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| 8 路 GPIO 直读 | 速度最快；如果同端口可近似同时读；代码最简单 | 占 8 个 IO；当前会和旋钮 `PB25/PB18/PB21` 冲突 | IO 足够、追求最简单硬件 |
| 74HC4051 扫描 | 只占 4 到 5 个 IO；采样确定性高；几十微秒内可扫完 8 路 | 8 路不是完全同时采样；要考虑 4051 导通电阻、传感器输出阻抗和电平范围 | 原始 8 路数字循迹，IO 不够时优先推荐 |
| UART 串口传输 | 线少；可以让外部小 MCU 先滤波/打包/校验 | 真 UART 有波特率、帧格式和调度延迟；115200 下 1 字节约 86.8 us，还会有协议开销 | 循迹板本身带 MCU，并已经输出协议帧 |

结论：如果你的 8 路循迹只是 8 根原始数字电平，74HC4051 比“硬凑成 UART”更合适；如果你的循迹模块内部已经有单片机，能通过 `TX/RX` 主动发 `raw_bits` 或位置帧，那才适合走 UART。JY61P 也需要 UART，所以当前工程保留 UART0 给 JY61P，循迹走 4051。

## 上板注意

- 如果循迹板输出是 5V，而 4051 和 MSPM0 运行在 3.3V，必须做分压、限流或电平转换。
- 如果黑线输出高电平，把 `g_line_sensor.active_level` 改为 `LINE_TRACE_ACTIVE_HIGH`。
- 如果左右顺序反了，优先调整 `g_line_sensor.channel_order` 或 4051 的 `Y0..Y7` 线序，不要改算法核心。
- 真实跑车前保持 `BOARD_APP_ENABLE_MOTOR_OUTPUT=0`，先用 SWD/串口/LCD 观察 `line_raw_bits` 和 `line_active_bits` 是否随遮挡正确变化。
